## MainActivity 核心流程

### 1. 初始化

```kotlin
val app = application as DailyTasksApp
repository = app.repository   // 从 Application 拿全局 Repository
```

### 2. 订阅任务列表（响应式 UI）

```kotlin
lifecycleScope.launch {
    repository.observeTasks().collect { tasks ->
        adapter.submitList(tasks)      // 刷新列表
        updateEmptyState(tasks)        // 空状态提示
        updateProgress(tasks)          // 更新 "已完成 x/y"
    }
}
```

**关键概念**：`collect` 会在数据库每次变化时被调用，不需要手动刷新列表。

### 2b. 检查更新（v1.3）

冷启动时在后台检查 GitHub 最新 Release，有新版本则弹窗；详见 [07-在线更新.md](07-在线更新.md)。

```kotlin
if (savedInstanceState == null) {
    UpdateManager.checkOnLaunch(this)
}
```

### 3. 增删改任务

| 操作 | 触发 | 调用 |
|------|------|------|
| 添加 | FAB 按钮 | `repository.addTask(title)` |
| 编辑 | 铅笔图标 | `repository.updateTitle(id, title)` |
| 删除 | 垃圾桶（有二次确认） | `repository.deleteTask(id)` |
| 打钩 | 点击行或 CheckBox | `repository.toggleCompleted(id)` |

每次数据变更后调用 `refreshWidget()`，让桌面小组件同步更新。

### 4. 重置相关

| 按钮 | 行为 |
|------|------|
| 修改重置时间 | `saveResetTime()` + `ResetScheduler.schedule()` |
| 立即重置 | `resetAllTasks()` |

## TaskAdapter.kt — 列表每一项

`RecyclerView.Adapter` 的标准用法：

- `onCreateViewHolder`：加载 `item_task.xml` 布局
- `onBindViewHolder`：把 `TaskEntity` 数据填到视图上
- `ListAdapter` + `DiffUtil`：高效对比列表变化，只更新变动的行

已完成任务的视觉处理：

```kotlin
binding.title.paintFlags or Paint.STRIKE_THRU_TEXT_FLAG  // 删除线
binding.title.alpha = 0.5f                                // 半透明
```

## ViewBinding

```kotlin
binding = ActivityMainBinding.inflate(layoutInflater)
binding.recyclerView.adapter = adapter
```

`ActivityMainBinding` 由 Android 根据 `activity_main.xml` 自动生成，替代 `findViewById`。
