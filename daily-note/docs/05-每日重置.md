# 05 - 每日重置

每天到点（默认 08:00）自动把所有任务变回「未打钩」。

## 涉及文件

| 文件 | 作用 |
|------|------|
| `ResetScheduler.kt` | 注册下一次闹钟 |
| `receiver/DailyResetReceiver.kt` | 闹钟响时执行重置 |
| `receiver/BootReceiver.kt` | 开机后补检查 |
| `TaskRepository.ensureDailyReset()` | 打开 App 时补检查 |

## 重置的三种触发方式

```
┌─────────────────────────────────────────────┐
│  1. AlarmManager 到点触发（主要方式）          │
│  2. 开机后 BootReceiver 补检查                │
│  3. 打开 App 时 ensureDailyReset() 兜底       │
└─────────────────────────────────────────────┘
```

### 1. 闹钟（ResetScheduler）

```kotlin
// 计算下一次触发时间
var trigger = 今天的 resetHour:resetMinute
if (trigger 已过) trigger = 明天的同一时间

alarmManager.setExactAndAllowWhileIdle(RTC_WAKEUP, triggerAtMillis, pendingIntent)
```

类似手机闹钟：改时间后重新 `schedule()`，旧闹钟被覆盖，按新时间响。

**举例**：原 8:00，7:00 改成 9:00 → 8:00 不响，9:00 重置。

### 2. 开机补重置（BootReceiver）

手机关机错过闹钟时，开机后：

```kotlin
repository.ensureDailyReset()  // 检查是否跨天且过了重置时间
ResetScheduler.schedule()      // 重新注册下一次闹钟
```

### 3. 打开 App 兜底（ensureDailyReset）

```kotlin
if (上次重置日期 < 今天) {
    if (隔了多天 || 当前时间 >= 重置时间) {
        resetAllTasks()  // 清空打钩，更新 lastResetDate
    }
}
```

## 改重置时间 vs 立即重置

| 操作 | 行为 |
|------|------|
| 修改重置时间 | 只改闹钟，**不**清空打钩 |
| 立即重置按钮 | **马上**清空所有打钩 |

## DailyResetReceiver 流程

```
闹钟触发
  → resetAllTasks()        // 数据库 isCompleted 全变 false
  → refreshAll()           // 刷新小组件
  → schedule()             // 注册明天的闹钟
```

`goAsync()` + `CoroutineScope`：广播接收器里做数据库操作必须在后台线程，完成后 `pendingResult.finish()`。

## AndroidManifest 注册的权限

```xml
<uses-permission android:name="android.permission.RECEIVE_BOOT_COMPLETED" />
<uses-permission android:name="android.permission.SCHEDULE_EXACT_ALARM" />
```

Android 12+ 精确定时闹钟可能需要用户在系统设置里允许「闹钟与提醒」。

## 学习总结

本 App 涵盖了 Android 开发中几个常见模式：

1. **Room + Repository** — 本地数据
2. **Flow + Coroutines** — 响应式 UI
3. **App Widget + RemoteViews** — 桌面组件
4. **AlarmManager + BroadcastReceiver** — 定时任务
5. **Application 单例** — 全局共享 Repository
