# 02 - 数据层

数据层负责**任务的持久化**和**重置相关的设置**，核心在 `data/` 目录。

## 三层结构

```
TaskEntity（数据类）
      ↓
TaskDao（数据库操作接口）
      ↓
AppDatabase（Room 数据库单例）
      ↓
TaskRepository（业务封装，对上层提供 API）
```

## TaskEntity.kt — 一条任务长什么样

```kotlin
@Entity(tableName = "tasks")
data class TaskEntity(
    @PrimaryKey(autoGenerate = true) val id: Long = 0,  // 自增主键
    val title: String,          // 任务文案
    val sortOrder: Int,         // 排序（按添加顺序递增）
    val isCompleted: Boolean,   // 今天是否已完成
)
```

`@Entity` 告诉 Room：这个类对应数据库里的一张表 `tasks`。

## TaskDao.kt — 数据库 CRUD

DAO = Data Access Object，定义所有 SQL 操作：

| 方法 | 作用 |
|------|------|
| `observeAll()` | 返回 `Flow`，数据变化时 UI 自动更新 |
| `getAll()` | 一次性取全部（小组件用） |
| `insert()` | 新增任务 |
| `update()` | 更新任务（改标题、改完成状态） |
| `deleteById()` | 删除 |
| `resetAllCompleted()` | 把所有 `isCompleted` 设为 false |

`Flow` 是 Kotlin 的响应式数据流：数据库一变，订阅者就会收到新列表。

## AppDatabase.kt — 数据库单例

```kotlin
Room.databaseBuilder(context, AppDatabase::class.java, "daily_tasks.db").build()
```

使用**单例模式**：整个 App 只开一个数据库连接，避免重复创建。

## TaskRepository.kt — 业务逻辑中枢

上层（MainActivity、小组件）**不直接访问 DAO**，而是通过 Repository：

| 方法 | 谁调用 | 做什么 |
|------|--------|--------|
| `addTask()` | MainActivity | 校验空文案、15 条上限，然后插入 |
| `toggleCompleted()` | MainActivity、小组件 | 切换打钩状态 |
| `ensureDailyReset()` | MainActivity、开机 | 跨天且过了重置时间 → 自动清空打钩 |
| `saveResetTime()` | MainActivity | 只保存时间，不立即重置 |
| `resetAllTasks()` | 立即重置按钮、闹钟 | 清空所有打钩 |

### SharedPreferences 存的设置

```kotlin
KEY_RESET_HOUR / KEY_RESET_MINUTE  // 每日重置时间，默认 8:00
KEY_LAST_RESET_DATE                // 上次重置的日期，如 "2026-07-03"
```

## 学习要点

1. **Room 三件套**：Entity + Dao + Database
2. **Repository 模式**：UI 不碰 SQL，统一走 Repository
3. **Flow + Coroutines**：`observeTasks()` 配合 `lifecycleScope.launch { collect { } }` 实现自动刷新
