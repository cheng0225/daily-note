# 04 - 桌面小组件

小组件 = 桌面上的「透明便签」，不用打开 App 就能看任务、打钩。

## 涉及文件

| 文件 | 作用 |
|------|------|
| `widget/TaskWidgetProvider.kt` | 小组件入口，系统回调 |
| `widget/TaskWidgetService.kt` | 列表数据工厂（RemoteViewsService） |
| `widget/WidgetToggleReceiver.kt` | 处理点击打钩 |
| `res/layout/widget_task_list.xml` | 小组件整体布局 |
| `res/layout/widget_task_item.xml` | 每一条任务的布局 |
| `res/xml/task_widget_info.xml` | 小组件尺寸、预览等配置 |

## 透明样式怎么做的

桌面上的小组件**背景完全透明**，直接透出壁纸：

```xml
<!-- widget_task_list.xml -->
android:background="@color/widget_bg"   <!-- colors.xml 里定义为 transparent -->
```

文字在壁纸上可能看不清，所以加了**浅色阴影**：

```xml
android:textColor="@color/widget_text_primary"    <!-- 半透明黑 -->
android:shadowColor="@color/widget_text_shadow"  <!-- 白色阴影 -->
android:shadowRadius="3"
```

| 颜色 | 值 | 用途 |
|------|-----|------|
| `widget_text_primary` | `#DE000000` | 未完成 task 文字 |
| `widget_text_secondary` | `#99000000` | 已完成 task（更淡） |
| `widget_bg` | `transparent` | 背景透明 |

> **注意**：部分国产 ROM（如 vivo）可能给小组件加一层系统底板，透明效果因机型而异。

### 预览 vs 实际

- **桌面实际**：`widget_task_list.xml`（透明）
- **选择小组件时的预览**：`widget_task_list_preview.xml`（带渐变假壁纸，方便你看效果）

## 小组件工作原理

Android 小组件运行在**独立进程**，不能直接用普通 View，要用 `RemoteViews`（跨进程 UI 描述）。

```
系统桌面
   │ 定时/数据变化时调用
   ▼
TaskWidgetProvider.onUpdate()
   │ 构建 RemoteViews
   ├── ListView ← TaskWidgetService（提供列表数据）
   ├── 标题点击 → 打开 MainActivity
   └── 列表项点击 → WidgetToggleReceiver（打钩）
```

### 列表数据：RemoteViewsService

普通 `ListView` 在 Activity 里用 Adapter；小组件里用 `RemoteViewsService` + `RemoteViewsFactory`：

```kotlin
override fun onDataSetChanged() {
    tasks = runBlocking { app.repository.getTasks() }  // 从数据库读
}

override fun getViewAt(position: Int): RemoteViews {
    // 为每一行构建 RemoteViews
}
```

### 点击打钩：PendingIntent 模板

列表里每一行的点击不能各自注册 Intent，要用**模板 + 填充**模式：

```kotlin
// Provider 里设置模板
views.setPendingIntentTemplate(R.id.widget_list, togglePendingIntent)

// Factory 里为每行填充 taskId
views.setOnClickFillInIntent(R.id.widget_item_row, fillIntent)
```

点击后 → `WidgetToggleReceiver` → `repository.toggleCompleted()` → `refreshAll()` 刷新小组件。

## 刷新小组件的时机

| 时机 | 调用 |
|------|------|
| App 内改数据 | `MainActivity.refreshWidget()` |
| 小组件点击打钩 | `TaskWidgetProvider.refreshAll()` |
| 每日重置闹钟 | `DailyResetReceiver` 里刷新 |
| 开机 | `BootReceiver` 里刷新 |
