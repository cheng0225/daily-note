# 07 - 在线更新

v1.3 起支持通过 **GitHub Release** 检查并下载新版本。代码在 `update/` 目录，`MainActivity` 冷启动时触发检查。

## 用户流程

```
打开 App（冷启动）
    ↓
后台请求 GitHub API：最新 Release
    ↓
┌─ 无新版本 / 连不上 GitHub ─→ 静默跳过，正常使用
│
└─ 有新版本
       ↓
   弹窗：「发现新版本 vX.X，是否立即更新？」
       ├── 稍后 → 记录该版本号，不再提示（直到下一个新版本）
       └── 立即更新
              ↓
          App 内下载 APK（进度条）
              ├── 成功 → 调系统安装器 → 用户点「安装」
              └── 失败 → 提示「无法连接 GitHub，请开启代理后重试」
```

**说明**：

- 下载和检查都在 App 内完成，**不跳转浏览器**
- Android 不允许静默安装，最后一步仍需用户在系统界面点「安装」
- App **不会**自动配置代理；国内连不上 GitHub 时需用户在系统或 v2rayN 里开代理

## 目录与职责

```
update/
├── UpdateConfig.kt        # GitHub 仓库地址、常量
├── GithubReleaseClient.kt # 调 API，解析 Release 与 APK 地址
├── VersionUtils.kt        # 比较版本号（如 1.2 vs 1.3）
├── ApkDownloader.kt       # HTTP 下载 APK 到 cache
├── ApkInstaller.kt        # FileProvider + 调系统安装器
└── UpdateManager.kt       # 弹窗、进度、串联全流程
```

## GitHub API

请求地址（见 `UpdateConfig.API_URL`）：

```
GET https://api.github.com/repos/cheng0225/daily-note/releases/latest
```

从响应中读取：

| 字段 | 用途 |
|------|------|
| `tag_name` | 版本 tag，如 `v1.3` |
| `assets[].name` | 找以 `.apk` 结尾的文件 |
| `assets[].browser_download_url` | APK 下载地址 |

发布 Release 时，APK 命名须与构建脚本一致：

```
daily-note-v{versionName}-debug.apk
```

例如 `daily-note-v1.3-debug.apk`，tag 为 `v1.3`。

## 版本比较

`VersionUtils.isNewer(remote, local)` 按 `.` 分段比较数字：

```kotlin
VersionUtils.isNewer("1.3", "1.2")  // true
VersionUtils.isNewer("1.2", "1.2")  // false
```

本地版本来自 `BuildConfig.VERSION_NAME`（与 `app/build.gradle.kts` 中 `versionName` 一致）。

## 跳过提示

用户点「稍后」后，将 `tag_name`（如 `v1.3`）写入 SharedPreferences（`update_prefs` / `skipped_version`）。同一版本再次打开 App 不再弹窗；发布 `v1.4` 后会重新提示。

## 安装 APK

Android 7+ 不允许 `file://` 直接安装，需 **FileProvider**：

1. `AndroidManifest.xml` 注册 `${applicationId}.fileprovider`
2. `res/xml/file_paths.xml` 暴露 `cache-path`（APK 下载到 `cacheDir`）
3. `ApkInstaller.install()` 用 `ACTION_VIEW` + `application/vnd.android.package-archive`

Android 8+ 还需 `REQUEST_INSTALL_PACKAGES` 权限；未授权时引导用户到「安装未知应用」设置页。

## MainActivity 中的调用

仅在**冷启动**（`savedInstanceState == null`）且**本次进程未检查过**时触发：

```kotlin
if (savedInstanceState == null) {
    UpdateManager.checkOnLaunch(this)
}
```

检查在 `lifecycleScope` 协程中异步执行，不阻塞界面加载。

## 权限

| 权限 | 用途 |
|------|------|
| `INTERNET` | 访问 GitHub API、下载 APK |
| `REQUEST_INSTALL_PACKAGES` | Android 8+ 安装未知来源 APK |

## 发布新版本时注意

1. 在 `app/build.gradle.kts` 递增 `versionCode` 和 `versionName`
2. 构建 APK：`gradlew assembleDebug` → 自动复制到 `releases/`
3. 用 `scripts/publish-github.ps1` 推送到 GitHub 并创建 Release，**上传对应 APK**
4. 用户需至少手动安装一次带更新功能的版本（如 v1.3），之后才能在 App 内更新
