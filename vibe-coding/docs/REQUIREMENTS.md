# 需求说明

## 1. 项目背景

开发一款**仅个人使用**的 Android 桌面小工具，体验类似 vivo 原子笔记的桌面便签：任务常驻桌面，随手勾选，每天自动「换新的一天」。

## 2. 用户故事

| 编号 | 作为用户，我希望… | 以便… |
|------|-------------------|--------|
| US-01 | 在桌面添加一个小组件 | 不用打开 App 就能看到今日任务 |
| US-02 | 点击任务旁的勾选框 | 标记该项已完成 |
| US-02b | 在 App 内同样能打钩 / 取消打钩 | 不打开桌面小组件也能更新进度 |
| US-03 | 每天固定时间（如 08:00）自动重置 | 第二天所有任务恢复未打钩，文案保留 |
| US-04 | 在 App 内编辑任务列表 | 随时调整今天要做什么 |
| US-05 | 在 App 内设置重置时间 | 按自己的作息定制「新的一天」 |

## 3. 功能需求

### 3.1 桌面小组件（P0 — 必须有）

- 支持添加到系统桌面（Launcher Widget）
- 展示**今日任务列表**，每项包含：
  - 任务文案
  - 完成状态（未打钩 / 已打钩）
- 在小组件上**直接点击**切换完成状态（无需进入 App）
- 已完成的任务视觉上区分（如删除线、灰色、勾选图标）
- 小组件尺寸：至少支持 **2×2** 或 **4×1** 等常见规格（具体尺寸实现阶段再定）

### 3.2 任务管理（P0）

- **首次安装任务列表为空**，不预置任何示例任务
- 用户自行维护任务列表，支持**添加、编辑、删除**任务文案（在 App 内完成，MVP 不要求小组件内编辑）
- 每日重置**仅清空完成状态**，不删除、不修改用户设置的任务文案
- 任务按**添加顺序**排列，MVP 不支持拖拽排序
- 任务数量上限 **15 条**，超出时提示并禁止继续添加
- 无任务时：App 内显示空状态引导添加；小组件显示「暂无任务」等占位文案

### 3.3 每日自动重置（P0）

- 用户可设置**每日重置时间**（时 + 分），默认 `08:00`
- 到达该时刻后：
  - 所有任务恢复为**未完成**
  - 小组件 UI 同步刷新
- **修改重置时间**：仅重新预约下一次闹钟，**不立即重置**（需立刻清空请点「立即重置」）。例如原 8:00、7:00 改成 9:00 → 8:00 不再触发，9:00 才重置
- 重置逻辑以**本地时区**为准
- 若该时刻手机关机/未唤醒，开机后应尽快补执行一次重置（若当日尚未重置）

### 3.4 设置页（P0 — MVP 必须有）

- 任务列表的添加、编辑、删除
- 设置每日重置时间
- 关于 / 版本信息（可选）

### 3.5 在线更新（P1 — v1.3 已实现）

- 打开 App 时检查 [GitHub Release](https://github.com/cheng0225/daily-note/releases) 是否有新版本
- 有新版本：弹窗提示，用户可「立即更新」或「稍后」
- **立即更新**：App 内下载 APK → 调系统安装器（用户仍需点「安装」）
- 连不上 GitHub：下载失败时提示用户开启代理；启动时检查失败则静默跳过
- 发布要求：Release 附带 `daily-note-v{版本}-debug.apk`，tag 如 `v1.3`
- 实现文档：[`daily-note/docs/07-在线更新.md`](../../daily-note/docs/07-在线更新.md)

### 3.6 暂不纳入 MVP

- 云同步、多设备
- 多组任务 / 多便签
- 提醒通知（到点提醒做某任务）
- 历史记录与统计
- 锁屏小组件

### 3.7 后续再考虑

- 小组件与应用**视觉样式**（配色、圆角、半透明等）— MVP 使用系统默认 / 简约样式即可

## 4. 交互说明

### 4.1 小组件

```
┌─────────────────────────┐
│  今日任务          3/5  │  ← 可选：完成进度
├─────────────────────────┤
│  ☐ 晨间阅读             │  ← 点击整行或勾选框切换状态
│  ☑ 回复邮件      (灰)   │
│  ☐ 运动 30 分钟         │
│  ☐ 写日报               │
└─────────────────────────┘
```

- 点击任务行 → 切换 ☐ / ☑
- 长按小组件 → 系统「移除 / 调整大小」等标准行为
- 若任务过多：小组件内滚动或「+N 项」省略（实现时二选一）

### 4.2 App 主界面

- 首次打开：空列表 + 引导「添加第一条任务」
- 任务列表：每条可点击打钩 / 取消打钩，可编辑文案、删除
- 底部或顶部：「添加任务」（已达 15 条时禁用并提示）
- 设置项：「每日重置时间」时间选择器
- 变更后立即刷新已放置的小组件

## 5. 数据需求

### 5.1 需要持久化的数据

| 字段 | 类型 | 说明 |
|------|------|------|
| 任务 ID | String / Long | 唯一标识 |
| 任务文案 | String | 显示内容 |
| 排序 | Int | 列表顺序 |
| 今日是否完成 | Boolean | 当日勾选状态 |
| 上次重置日期 | LocalDate | 用于判断是否需补重置 |
| 每日重置时间 | LocalTime | 默认 08:00 |

### 5.2 存储方式

- 本地存储即可（SharedPreferences + JSON，或 Room）
- 无需账号；**在线更新**时仅访问 GitHub 下载 APK，不上传用户数据

## 6. 非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | 小组件点击后 1 秒内更新 UI |
| 电量 | 重置使用 `AlarmManager` / `WorkManager`，避免常驻后台 |
| 兼容性 | 适配主流国产 ROM 桌面（小米、华为、OPPO、vivo 等） |
| 隐私 | 数据仅存本机，无上传 |
| 维护 | 个人项目，代码结构清晰即可 |
| 更新 | GitHub Release 分发；App 内检查并下载安装（v1.3+） |

## 7. 验收标准（MVP）

1. ✅ 能添加桌面小组件并显示任务列表  
2. ✅ 在小组件上点击可打钩 / 取消打钩，状态重启 App 后仍保留  
3. ✅ 在 App 内可编辑任务、修改重置时间  
4. ✅ 到达设定时刻后，所有任务自动变为未打钩  
5. ✅ 关机错过重置时间后，开机当日能正确补重置  

## 8. 已确认决策

| 项 | 决策 |
|----|------|
| 任务内容 | 用户自行添加、编辑、删除；每日重置只恢复未打钩，不改任务文案 |
| 首次安装 | 任务列表为空，无预置数据 |
| 打钩位置 | 桌面小组件 + App 内均可操作，状态同步 |
| 任务顺序 | 按添加顺序，MVP 不支持拖拽 |
| 任务上限 | 最多 15 条 |
| 修改重置时间 | 仅改闹钟时间，下次按新时间触发；立即重置用「立即重置」按钮 |
| 视觉样式 | MVP 用默认简约样式，美观与主题后续迭代 |
| 应用名称 | **每日便签** |
| 文档目录 | `vibe-coding/`（仅需求与规划） |
| 代码工程 | `daily-note/`（Android Studio 打开此目录） |
| 应用包名 | `com.vibecoding.dailytasks` |
| 在线更新 | GitHub Release API + App 内 APK 安装（v1.3） |
